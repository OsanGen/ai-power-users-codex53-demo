Lesson Diagram V2 Component

Purpose:
- Replace legacy lesson slide doodles with a semantic, mode-specific neural diagram engine.
- Render concept-accurate visuals for lesson/death-lesson cards with staged signal flow.
- Keep diagram quality high while preserving runtime safety through hard fallback to legacy canvas rendering.
- Execute a typography-first polish pass for all neural diagram modes without changing graph topology or learning logic, so the text system remains the controllable visual boundary for OSAN-brand compliance.

Inputs:
- `visualMode` string from narrative teach/death cards (`network_basic`, `inputs_nodes`, `weights_knobs`, `sum_bias`, `activation_gate`, `layers_stack`, `loss_meter`, `backprop_arrows`, `generalize_explain`).
- Diagram target rect (`x`, `y`, `w`, `h`) and accent color from `src/render.js`.
- Time (`game.globalTime`) and reduced-motion preference (`AIPU.input.prefersReducedMotion`) from runtime state.
- Optional libraries loaded by `index.html`:
  - PixiJS (`window.PIXI`)
  - ELKjs (`window.ELK` or `window.elk.ELK`)
  - pixi-particle-emitter (`PIXI.particles.Emitter` or equivalent global emitter constructor)

Outputs / API:
- `AIPU.lessonDiagramV2.init()`
  - Returns availability metadata for optional dependencies and schema version.
- `AIPU.lessonDiagramV2.renderTo(ctx, rect, mode, accent, time, reducedMotion)`
  - Returns `true` if V2 render succeeds and is painted onto target canvas context.
  - Returns `false` when unavailable/failed so caller can execute legacy fallback rendering.
- `AIPU.lessonDiagramV2.getDebugState()`
  - Returns deterministic diagnostics:
    - `mode`
    - `layoutVersion` (`manual` or `elk`)
    - `activeStage`
    - `activeLayoutSource` (`manual` or `elk`)
    - `activeRenderMode` (`canvas-fallback`, `auto`, `renderer-webgl`, etc.)
    - `legacyFallbackUsed`
    - `fallbackUsed`
    - dependency availability flags (`pixiAvailable`, `elkAvailable`, `emitterAvailable`)
    - last-failure details (`lastFailure`, `lastFailurePhase`, `lastFailureAt`, `failureCount`)
    - `schemaVersion`

- `Text-system contract (lesson-diagram-v2 only, render-facing only)`
  - `textPresets` and `SCENE_TEXT_PRESETS` define size/weight/contrast behavior for:
    - `nodePrimary`, `nodeSecondary`, `edgeChip`, `edgeTiny`, `heading`, `caption`
  - New node and edge metadata fields are optional and render-only:
    - `labelTone`, `labelLayout`, `labelPurpose`, `labelMaxWidth`
  - All label rendering now routes through `buildTextSpec()` -> `resolve*LabelPlacement()` -> `render*LabelCard()`.
  - No graph data model semantics were changed; only presentation metadata was added.

State / Edges:
- Layout cache is keyed by `mode + size bucket` and reuses cached placements.
- Manual layered layout is always available; ELK layout upgrades cache asynchronously when available.
- Layout normalization now fits node bounds (including node radii) and centers content inside the viewport, preventing top-left anchoring and excess dead space in tall diagram panels.
- Edge-label placement now de-conflicts crowded incoming edges by grouping labels per target node and applying deterministic slot offsets; `sum_bias` incoming labels (`w1/w2/w3/bias`) no longer stack on top of each other near `Σ`.
- Active layout source is recomputed from availability each render pass: manual is used while async ELK computes, then switched to ELK as soon as `elkPositions` resolves for that same mode/size key to prevent stale manual topology lingering after async completion.
- Additional viewport right padding and inward label anchors protect right-edge labels (`output` leg) from clipping in narrow or compact lesson cards.
- If PixiJS is missing or renderer creation fails, `renderTo()` uses deterministic 2D canvas scene drawing when possible and only falls back to legacy `render.js` sketch when a hard failure prevents V2 canvas rendering.
- If ELK is missing/fails, manual layout continues with deterministic structure.
- If particle emitter is missing/fails, spark fallback visuals are used and diagram remains functional.
- Stage animation is deterministic by mode with reduced-motion forcing stage `0` and minimal motion.
- Renderer resources are resized/reused for the current diagram surface and cleaned when invalidated.

Dependencies:
- Loaded in `index.html` before `src/render.js`:
  - `https://cdn.jsdelivr.net/npm/elkjs@0.10.0/lib/elk.bundled.js`
  - `https://cdn.jsdelivr.net/npm/pixi.js@7.4.3/dist/pixi.min.js`
  - `https://cdn.jsdelivr.net/npm/@pixi/particle-emitter@5.0.8/dist/particle-emitter.min.js`
- Consumed by `src/render.js` via `drawLessonSlideDiagram(...)`.

Acceptance checks:
- All nine `visualMode` scenes render topology-distinct diagrams (not simple recolors).
- `sum_bias` scene explicitly shows weighted inputs, `+b`, `Σ`, pre-activation node, activation node, and output.
- `sum_bias` edge labels around `Σ` are visually separated and readable at mobile and desktop panel widths.
- Graph content remains centered vertically/horizontally within the lesson diagram card; no large lower dead-zone caused by top anchoring.
- No visible “old layout then new layout” flash should occur while remaining on the same lesson card view.
- Diagram updates preserve existing lesson/death-lesson flow and input prompts (`Enter or Space` continue).
- `loss_meter`, `backprop_arrows`, and `generalize_explain` text labels stay under the established visual-noise budget and do not exceed readable density in dense scenes.
- `activation_gate`, `inputs_nodes`, and `weights_knobs` use compact lane-based edge text placement to avoid overlap at edge midpoints.
- Reduced-motion mode renders readable static diagrams with no pulse-driven motion dependency.
- When Pixi/ELK/emitter are unavailable, legacy `drawLessonSlideDiagramFallback()` path still draws with no runtime exceptions.
- `AIPU.render.renderGameToText()` includes `debug.lessonDiagram` state during lessons for deterministic automation checks.
- Deployment check for this component requires that live `index.html` includes `./src/lesson-diagram-v2.js` and the matching `v=` token; otherwise `AIPU.lessonDiagramV2` is unavailable and the legacy fallback can appear.
