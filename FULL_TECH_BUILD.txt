## 2) What this repo is
WONDERLAND is a browser-first vanilla web game (canvas-based), centered on a floor-progression shooter loop with HUD overlays, tutorial/lesson flow, upgrade progression, and share/replay-oriented UI controls. It ships as an interactive web app with static assets and QA hooks.

## 3) System map
- Components: `index.html`, `src/main.js`, `src/core.js`, `src/systems.js`, `src/render.js`, `src/audio.js`, `src/share.js`, `src/ui-tokens.js`, `src/content.js`, `src/upgrades.js`, `src/ui-tokens.js`.
- Boundaries: `index.html` loads modules; `core.js` wires shared globals; `main.js` owns loop/bootstrap/wiring; `systems.js` owns simulation/input/state transitions; `render.js` owns canvas rendering and FX pipeline; `audio.js` owns music/SFX + persistence; `share.js` owns modal/copy/share interactions.
- Key data flow: DOM/input/URL/state seed → simulation tick/update (systems) → draw pipeline (render) → overlay/audio/mute/share outputs.
- If DEV MODE is active, `final_build/BUILD_MAP.txt` is the canonical system map.

## 4) Entrypoints and run commands
- Local run: `python3 -m http.server 4173` then open `http://127.0.0.1:4173/index.html`.
- Build: none (static app).
- QA: `npm run qa:bootstrap`, `npm run qa:bootstrap:both`, `npm run qa:deps-contract`, `npx playwright test wonderland-qa.spec.js`, `node scripts/check-ui-bootstrap-oneshot.js`.
- Lint/format/typecheck: none centralized in package scripts; ad hoc checks are mostly Playwright + dependency contract checks.
- Environment notes: `QA_TARGET_URL`, `QA_STAGED_URL`, `QA_DEPLOYED_URL` for bootstrap routes.
