## 2) What this repo is
WONDERLAND is a browser-first vanilla JS canvas game that teaches neural-network concepts through floor progression, lesson overlays, and upgrade loops. It is a static web app with deterministic QA hooks (`window.advanceTime`, `window.render_game_to_text`) and script-driven UI-bootstrap checks.

## 3) System map
- Components and responsibilities:
  - `src/core.js`: shared state/input/constants, QA namespaces, and safety helpers.
  - `src/content.js`: narrative, floor definitions, song selection, upgrades copy, threat waves.
  - `src/audio.js`: floor-scoped background music playback and mute persistence.
  - `src/upgrades.js`: upgrade definition/selection/rules and derived stat aggregation.
  - `src/systems.js`: simulation, state transitions, input handling, collisions/spawn/kill, attack-disable and bomb gating.
  - `src/render.js`: draw pipeline, overlays, performance governor, visual effects, and debug state snapshot.
  - `src/lesson-diagram-v2.js`: semantic lesson diagram renderer with optional Pixi/ELK path + deterministic fallback.
  - `src/main.js`: bootstrap orchestration, RAF loop, and test hooks.
  - `src/share.js`: share modal lifecycle.
  - `index.html`: asset loading order and cache-bust tokens; optional CDN runtime dependencies.
- Key flows/boundaries:
  - `main.js` -> boots `systems`, `render`, and `share`; drives animation loop.
  - `systems` updates model state; `render` reads it for visuals and diagnostics.
  - narrative lesson/death content from `content.js` feeds both `systems` and `render` overlays.
  - lesson diagram rendering uses `render.drawLessonSlideDiagram()` -> `lessonDiagramV2.renderTo()` with debug payload fallback.
- If DEV MODE is active, `final_build/BUILD_MAP.txt` is the detailed architecture map.
